# 快速启动指南

## 方法一：使用安装脚本（推荐）

在终端中执行：

```bash
cd /Users/xuehao/Sites/oversea/tool/headcanon-generator-app
bash install.sh
```

## 方法二：手动安装

### 1. 确保 Node.js 已安装

```bash
node --version
npm --version
```

如果命令不存在，请先安装 Node.js：

```bash
# 使用 Homebrew（推荐）
brew install node

# 或者从官网下载
# https://nodejs.org/
```

### 2. 安装 pnpm（推荐）

```bash
npm install -g pnpm
```

或者：

```bash
brew install pnpm
```

### 3. 安装项目依赖

```bash
# 使用 pnpm（推荐）
pnpm install

# 或使用 npm
npm install
```

### 4. 配置环境变量

创建 `.env.local` 文件：

```bash
touch .env.local
```

编辑 `.env.local`，添加你的 SiliconFlow API key（可选，代码中已有默认值）：

```
SILICONFLOW_API_KEY=sk-your-api-key-here
SILICONFLOW_MODEL=deepseek-ai/DeepSeek-R1-0528-Qwen3-8B
```

**注意：** 如果不配置环境变量，项目会使用代码中的默认 API key，可以直接运行。

### 5. 启动开发服务器

```bash
# 使用 pnpm
pnpm dev

# 或使用 npm
npm run dev
```

### 6. 打开浏览器

访问 [http://localhost:3000](http://localhost:3000)

## 常见问题

### 问题：命令找不到 node 或 npm

**解决方案：**
1. 检查 Node.js 是否已安装：`which node`
2. 如果使用 Homebrew 安装，确保 PATH 包含 `/opt/homebrew/bin`
3. 在 `~/.zshrc` 或 `~/.bash_profile` 中添加：
   ```bash
   export PATH="/opt/homebrew/bin:$PATH"
   ```
4. 重新加载配置：`source ~/.zshrc`

### 问题：依赖安装失败

**解决方案：**
1. 清除缓存：`npm cache clean --force` 或 `pnpm store prune`
2. 删除 `node_modules` 和锁文件，重新安装
3. 检查网络连接

### 问题：缺少 SiliconFlow API Key

**解决方案：**
1. 项目已包含默认 API key，可以直接使用
2. 如需使用自己的 API key：
   - 访问 https://siliconflow.cn/
   - 注册/登录账号
   - 在控制台获取 API key
   - 将 key 添加到 `.env.local` 文件中
